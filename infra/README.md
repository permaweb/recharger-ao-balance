# recharger-ao-balance infra

Host `recharger-ao-balance` as a HyperBEAM node on a server.

The service starts HyperBEAM directly with `erl`, using:

- `config.json`
- `_build/preloaded-store`
- the preload index generated by `rebar3 device preload`

## Build

From the repo root:

```sh
rebar3 compile
rebar3 device preload
```

## Install

```sh
sudo useradd --system --home /opt/recharger-ao-balance --shell /usr/sbin/nologin hyperbeam || true
sudo install -d -o hyperbeam -g hyperbeam /opt/recharger-ao-balance
sudo cp -a . /opt/recharger-ao-balance/
sudo chown -R hyperbeam:hyperbeam /opt/recharger-ao-balance
sudo install -m 0644 infra/recharger-ao-balance.service /etc/systemd/system/recharger-ao-balance.service
sudo install -m 0644 infra/recharger-ao-balance.env.example /etc/default/recharger-ao-balance
sudo install -m 0755 infra/start-recharger-node.sh /opt/recharger-ao-balance/infra/start-recharger-node.sh
sudo systemctl daemon-reload
sudo systemctl enable --now recharger-ao-balance
```

## Check

```sh
systemctl status recharger-ao-balance
journalctl -u recharger-ao-balance -f
```

Expected endpoint:

```text
http://<server>:18735
```

## Bundler gating

`config.json` gates bundler uploads through P4 with:

- `ledger-device`: `recharging-ledger@1.0`
- `pricing-device`: `metering@1.0`
- `metering-rates.arweave-bytes`: `2`
- `metering-rates.beam-reductions`: `0`

`dev_bundler` meters the serialized ANS-104 item size, then P4 charges the
request signer from the recharging ledger after the upload succeeds.
